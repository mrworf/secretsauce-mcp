# SecretSauce Security Notes

- The gateway is not a secrets vault. It is a service-scoped HTTP gateway that keeps configured credentials out of MCP tool results.
- Policy is enforced before credential substitution.
- Destination scheme, host, and port are validated before credential substitution.
- Caller-controlled authority, forwarding, and hop-by-hop headers are rejected before credential substitution, and the outbound HTTP authority is derived from the validated destination URL.
- Configured-credential `gref_…` values are bound to subject, service, destination, and credential. Response-derived `sec_…` values are bound to subject and service so they can be reused across that service's destinations. MCP HTTP is stateless, and every POST is authenticated independently.
- Stateless MCP transport does not make runtime capability state stateless. The supported topology is exactly one gateway replica: references, denials, and related state are instance-local, so horizontal load balancing causes intermittent reference failures. Sticky sessions are not a substitute for an unimplemented shared atomic capability store.
- Named header credentials may opt into `usage.enforce: true`, which makes the header gateway-owned and restricts its reference to the configured prefix/name/suffix template. Enforcement is off by default; unenforced references retain flexible substring substitution. Repairable enforced-header overrides are clobbered with sanitized warnings, while missing, ambiguous, and wrongly placed references fail before downstream I/O.
- External OAuth requires one configured non-empty string principal claim (default `sub`); identity-less tokens are rejected rather than mapped to a shared subject.
- `tls.verify: false` is supported for self-signed homelab services and is reported in response metadata and audit events.
- Secretlint scans response header values and UTF-8 body source text. A separate case-insensitive regex catalog protects sensitive response header names and complete string values in tolerant JSON/environment shapes even when endpoint Secretlint is disabled. A gateway-owned detector also protects canonical HTTP Basic credentials with non-empty usernames and passwords regardless of their surrounding names or Secretlint settings. JSON is never deserialized or reserialized, so all bytes outside replacement ranges remain unchanged.
- Exact configured source values are always tokenized, even when endpoint Secretlint rules are disabled. Usage prefixes and suffixes are non-secret request syntax and are not part of credential identity.
- `gref_` and `sec_` are reserved prefixes. A candidate is exempted only when it is live and belongs to the current subject/service; forged, expired, or wrong-scope candidates are wrapped in a new `sec_…` and audited without their values.
- A whole response body is Base64-decoded only when declared by `Content-Transfer-Encoding: base64`; decoded data must be UTF-8 and is re-encoded canonically after scanning. A string request body with the same declaration is decoded for opaque-token substitution and canonically re-encoded before delivery. JSON request strings are edited only at complete token substrings and restored secrets are JSON-escaped; malformed or key-position tokens fail before downstream I/O. Field-level Base64/PEM is not decoded, and undeclared Base64-looking content remains opaque.
- Response scanning is defense in depth, not a sandbox for arbitrary downstream computation. An approved endpoint that can echo, encode, template, execute, inspect, or otherwise transform caller-controlled input may return a reversible representation of a substituted credential that no finite pattern scanner can reliably identify.
- Treat allowed methods and routes as part of the credential security boundary. Prefer narrowly structured non-computational APIs, and avoid generic debug, echo, script, command, template, and proxy endpoints unless the residual credential-exfiltration risk is explicitly accepted.
- Cookie headers are rejected on proxied requests and removed from proxied responses. Cookie-dependent downstream APIs are unsupported.
- Request and returned response `Content-Length` values are computed from final transmitted/returned bytes after substitution or tokenization. Caller-supplied transfer encoding is rejected.
- Scanning uses a bounded worker pool and fails closed on overload, timeout, an incomplete recognized sensitive string, or scanner failure. Other recoverable JSON damage is retained when every sensitive finding has a safe source range.
- Downstream responses are aborted at the configured byte limit and fail closed without returning partial content.
- MCP POST authentication occurs before bounded body parsing; declared and streamed inbound sizes are both enforced.
- Avoid allowing endpoints that return backups, complete config dumps, token lists, private keys, or other bulk secret material.
- Audit events are sanitized centrally before memory retention or JSONL writing. Caller-controlled text is checked for exact configured credentials, opaque reference candidates, canonical HTTP Basic credentials, and a bounded catalog of common credential patterns. Audit events do not include raw credentials, opaque reference values, Authorization headers, cookies, request bodies, or response bodies. Pattern scanning remains defense in depth rather than a universal secret classifier.
- Durable audit open or write failure is fail-open for privileged operations but makes readiness fail with a path-free, sticky audit-degraded check until restart. Operators must monitor `/health` and audit storage capacity.
- The single replica should use stable read-only OAuth signing-key storage and writable persistent audit storage. Built-in OAuth refresh persistence is hash-only and single-writer; opaque gateway references remain intentionally ephemeral across restarts.
- Debug logging is opt-in through `logging.level: debug` and records only sanitized structural details and tokenization counts.
- Database-managed policy evaluates normalized methods, canonical destination hosts, and the exact canonical pathname through one pure evaluator. Greatest priority wins, deny wins equal-priority ties, and service plus every used credential boundary must independently allow.
- Policy simulation performs no downstream I/O and returns only structured reason codes and authority-filtered links. Safe copy excludes credential/vault/runtime material; cross-service copies are disabled and unassigned, and atomic bulk-copy retries are bound to a durable non-secret batch UUID.
